title: "API error responses and status codes"
description: "PathForge.ai returns structured JSON errors with HTTP status codes. Learn what each error means and how to handle 400 and 500 responses in your integration."
---

When a request to `POST /api/generate-roadmap` cannot be completed, the server returns a JSON object with a single `error` field describing what went wrong. Your client should always check the HTTP status code before attempting to parse `data` from the response body.

## Error response format

Every error — regardless of status code — follows the same shape:

```json
{ "error": "Error message string" }
```

There is no `data` field in an error response.

## Error reference table

| Status | Error message | Cause | Resolution |
|--------|---------------|-------|------------|
| `400` | `"All fields are required."` | One or more of the five required body fields are missing or empty. | Ensure all five fields — `name`, `educationLevel`, `interests`, `currentSkills`, and `careerGoal` — are present and non-empty in the request body. |
| `500` | `"AI generation failed. Please check your API key and try again."` | The Groq API call failed due to an invalid key, exceeded quota, or a network error. | Verify that `GROQ_API_KEY` is correctly set and that your Groq account has remaining quota. |
| `500` | `"Failed to parse AI response. Please try again."` | The AI returned a response that could not be parsed as valid JSON. | This is a transient error. Retry the request — the next attempt will typically succeed. |

## Handling errors in code

The example below shows how to distinguish between a successful response and an error response, and how to surface the error message in your UI:

```javascript
try {
  const response = await fetch('https://pathforge-ai-api.onrender.com/api/generate-roadmap', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(profileData)
  });

  if (!response.ok) {
    const errorData = await response.json();
    console.error('PathForge error:', errorData.error);
    // Handle error in UI
    return;
  }

  const result = await response.json();
  // Use result.data
} catch (networkError) {
  console.error('Network error:', networkError);
}
```

<Tip>
  The `"Failed to parse AI response"` error is usually transient. For production integrations, implement a retry with exponential backoff — for example, retry up to three times with delays of 1 s, 2 s, and 4 s before surfacing the error to the user.
</Tip>
